LLM 비용 시뮬레이터 리뷰
직접 클릭해가며 살펴봤습니다. 컨셉과 구성 골격은 PM/개발자가 의사결정할 때 바로 쓸만한 방향인데, 세부 구현에서 몇 가지 치명적인 버그와 UX 이슈가 보입니다. 중요한 것부터 정리했습니다.
1. 치명적 버그 (먼저 고쳐야 함)
① "이주 비교" 카드의 수치가 고정값입니다.
모델을 Claude Haiku 4.5 → Claude Opus 4.7로 바꿔도, 후보 모델을 Gemini 3.1 Flash → GPT-5.4 nano로 바꿔도, 워크로드 프리셋을 바꿔도, 상단의 "월 158달러 / 연간 1,890달러", "월 6달러 / 연간 69달러" 숫자가 그대로 유지됩니다. 아래 시나리오 플래너와 요약 보고서는 정상 반영되는 것으로 보아, 이 카드 컴포넌트가 state를 구독하지 않고 초기값이나 별도 상수를 표시하고 있는 것 같습니다. "이주 비교"는 이 도구의 핵심 value proposition인데 숫자가 틀리면 전체 신뢰가 깨집니다. 최우선 수정 대상입니다.
② 캐시 적중률 라벨이 업데이트되지 않습니다.
슬라이더를 움직이면 값 자체는 요약 보고서/비용 계산에 반영되는데, 상단의 "캐시 적중률: 50 %" 라벨 텍스트는 50에서 멈춰 있습니다. 시나리오 플래너에서도 세 열 모두 "NaN %"로 표시됩니다. 아마 value는 state에 연결돼 있지만, 표시 문자열은 초기 렌더 시 한 번만 바인딩되거나, 시나리오 플래너에서는 파싱 전 값이 넘어가고 있는 것으로 보입니다.
③ 통화 단위 표기 불일치.
한 화면 안에서 "136달러", "$195", "832달러", "$1,634", "1,634달러"가 섞여 나옵니다. 시나리오 플래너에서는 같은 행/열에서도 포맷이 제각각입니다. 이사회 보고서가 타깃이라면 포맷은 한 가지로 통일되어야 설득력이 생깁니다.
2. 번역/로컬라이제이션 문제
한국어 번역 품질이 들쭉날쭉합니다. 대표적으로 "Claude Opus 4.7"이 드롭다운에서는 "클로드 오푸스 4.7"로 나오는데, 선택 후 "이주 비교" 카드에는 "클로드 작품 4.7"로 표시됩니다. Opus를 "작품"으로 기계 번역한 흔적입니다. 그 외에도 "배치 모드 ~에 / 끄다"("On/Off"를 번역한 듯), "교통"(아마 "Traffic" → 트래픽), "이주 비교"("Migration 비교" → 마이그레이션 비교), "나N %"("NaN") 등 자동 번역에 의존한 부자연스러운 표현이 많습니다. 그리고 코드 생성 프리셋을 선택하자 이사회 요약 보고서가 갑자기 영어로 바뀌었습니다 — 다국어 분기가 일관적이지 않게 동작하고 있습니다.
모델 라벨 자체도 제각각입니다: "1M당 $ 2.5 /$ 15", "100 만 개당 $ 0.2 / $ 1.25", "1 백만당 1 달러 / 5 달러", "100만 회당", "1M당 0.3 달러" 등 단위 표현이 전부 달라서 드롭다운이 혼란스럽습니다. 입력가 / 출력가 (단위: 1M tokens) 같은 형태로 통일하는 걸 권합니다.
3. UX/기능 측면 제안
"작업 부하 사전 설정"은 좋은 기능인데, 선택해도 어떤 값이 자동 세팅됐는지 사용자가 명시적으로 알 수 없어서 학습 곡선이 생깁니다. 프리셋 호버 시 tooltip으로 "기본 채팅: 입력 1억 / 출력 5천만, 캐시 20%" 같은 preview를 보여주면 좋겠습니다.
시나리오 플래너의 "최상/베이스/최악" 정의가 불투명합니다. 최상이 트래픽 -30%, 최악이 +100%라는 건 테이블을 읽고 나서야 알게 되는데, 이 규칙은 사용자가 직접 조절할 수 있어야 비즈니스 의사결정용 도구로 쓸 수 있습니다. 또한 캐시 적중률과 배치 모드가 시나리오별로 다르게 설정되는 로직도 현재는 잘 안 보입니다.
현재/후보 모델이 같은 모델을 선택할 수 있게 되어 있는데, 이 경우 "-96.3%" 같은 절감률이 의미 없게 나옵니다. 같은 모델 선택 시 "현재와 동일한 모델입니다" 안내를 띄우거나 후보에서 현재 모델을 비활성화하는 편이 낫습니다.
이사회 요약 보고서의 "복사" / "PNG 내보내기"는 좋은 포인트입니다. 다만 이사회용이라면 **출처 링크(공식 가격표 URL)**를 보고서에 같이 넣어주는 게 신뢰도를 크게 높입니다. 현재는 "공식 API 문서 기준"이라고만 쓰여 있어 검증이 불가합니다.
4. 접근성/입력 검증
월간 입력/출력 토큰 필드에 음수나 문자를 넣었을 때 어떻게 처리되는지, 그리고 너무 큰 수(예: 10조)를 넣어도 UI가 깨지지 않는지 확인이 필요합니다. 숫자 입력에 스텝/min/max 속성과 천단위 구분 표시(예: 100,000,000)가 없어서 0의 개수를 세야 합니다. 큰 수를 주로 다루는 도구이므로 이 부분은 반드시 개선하는 게 좋습니다.
총평
컨셉 ★★★★☆ — "마이그레이션 ROI + 시나리오 계획 + 이해관계자 내보내기"라는 포지셔닝은 PM이 벤더 비교 자료 만들 때 정확히 필요한 것입니다.
구현 완성도 ★★☆☆☆ — 상단 비교 카드가 하드코딩된 수치를 보여주고, 캐시 적중률 라벨이 동기화 안 되고, 통화/언어가 섞이는 것은 데모에서 바로 드러나는 문제입니다. 사내 PM에게 보여주기 전에 위 ①②③만이라도 먼저 고치는 것을 권합니다.
추천 우선순위: (1) 이주 비교 카드 state 연결 → (2) 캐시 적중률/NaN 표시 수정 → (3) 통화·언어 포맷 일원화 → (4) 모델 라벨 단위 통일 → (5) 번역 재검수(특히 "작품", "교통", "~에/끄다") → (6) 프리셋 preview 및 시나리오 파라미터 커스터마이징.
