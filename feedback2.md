LLM Cost Planner 사용성 체크 리포트
사이트를 직접 조작하면서 여러 케이스로 테스트해봤어요. 전체적으로는 UI가 깔끔하고 "Migration ROI · Scenario Planning · Stakeholder Export"라는 컨셉이 명확하게 전달되고 있어요. 다만 몇 가지 개선할 만한 포인트들이 보였습니다.
잘 된 점
구성(Configuration) → 비교(Migration Comparison) → 시나리오(Scenario Planner) → 요약(Board-Ready Summary)로 이어지는 세로 흐름이 직관적이에요. Workload Preset 버튼(Basic Chat, Code Generation 등)을 누르면 토큰 수, 캐시율, 배치 모드가 한 번에 세팅돼서 사용이 빠르고, Best/Base/Worst 시나리오를 색(초록·회색·빨강)으로 구분한 것도 한눈에 들어옵니다. 현재 모델과 후보 모델을 같은 모델로 맞췄을 때 "Current and candidate are the same model..."이라는 안내가 뜨는 에러 핸들링도 적절해요. 입력값(토큰 수)이나 슬라이더(캐시율), 체크박스(Batch Mode)를 변경하면 모든 카드가 실시간으로 갱신되는 반응성은 아주 좋습니다.
개선이 필요한 점
첫째, 입력값 검증이 약해요. Monthly Input Tokens 필드에 abc를 입력했더니 자동으로 0으로 바뀌는데, 사용자에게 "숫자만 입력 가능"이라는 피드백이 없어서 왜 바뀌었는지 알기 어려워요. 또 10,000,000,000(100억) 같은 비현실적으로 큰 값을 넣어도 그대로 계산해서 월 $32,375 같은 결과를 내보내는데, 상한선 경고나 안내가 있으면 좋겠습니다.
둘째, 모바일 반응형이 구현돼 있지 않아요. <meta viewport> 태그는 있지만 CSS 미디어 쿼리가 하나도 없어서, 좁은 화면에서 보면 카드·표·프리셋 버튼들이 모두 가로로 터져 스크롤을 강제하게 됩니다. 특히 Scenario Planner 표와 모델 선택 드롭다운은 모바일에서 매우 불편할 거예요.
셋째, 페이지 언어가 lang="ko"로 설정돼 있지만 본문은 전부 영어예요. 한국어 사용자 타겟이라면 UI 텍스트를 번역하거나, 영어 유지라면 lang="en"으로 맞추는 게 접근성·SEO 측면에서 맞습니다.
넷째, 모델 드롭다운이 가격과 함께 한 줄에 너무 많은 정보("GPT-5.4 — $2.50 / $15.00 per 1M tokens")를 담고 있어 긴 모델명이 잘려 보일 수 있어요. 벤더별 그룹화(optgroup)나 input/output 가격을 컬럼으로 분리해 테이블로 고르게 하는 방식도 고려해볼 만합니다.
다섯째, Cache Hit Rate 슬라이더에 숫자 입력 옵션이 없어서 정확한 값(예: 77%)을 맞추려면 마우스 드래그가 꽤 까다로워요. 슬라이더 옆에 숫자 입력 필드를 병행하면 사용성이 올라갑니다.
여섯째, Scenario Planner의 Best/Base/Worst 기준(예: Traffic −30%, Cache 80%)이 어디서 결정된 값인지 설명이 없어요. 헤더에 툴팁이나 "?" 아이콘으로 가정치를 조정하거나 최소한 설명하는 UI가 있으면 이해도와 신뢰성이 훨씬 올라갑니다.
일곱째, Board-Ready Summary 텍스트 자동 생성이 기본 톤으로만 나와서, 청중(임원/엔지니어)이나 톤(간단/상세)을 고를 수 있는 옵션이 있으면 "Stakeholder Export"라는 컨셉이 더 살 것 같아요. Copy 버튼 클릭 후 "복사되었습니다" 같은 토스트 피드백도 지금은 없어서 추가하면 좋겠습니다.
여덟째, 접근성 측면에서 프리셋 버튼들에 상태 표시가 약해요. 선택된 프리셋은 테두리만 살짝 바뀌는데, aria-pressed 속성과 함께 색상 대비를 더 높이면 키보드/스크린리더 사용자에게 더 친절해집니다. 또 큰 숫자(예: $32,375)만 색으로 구분하는 대신 아이콘(▲/▼)을 병행하면 색약 사용자에게도 좋아요.
아홉째, 데이터 투명성이 부족합니다. 하단에 "Prices based on official API docs · April 2026"이라고만 적혀 있는데, 각 모델 가격의 출처 링크나 업데이트 날짜를 모델별로 표기하면 이해관계자에게 제출하는 자료로 쓰기 더 좋겠습니다.
우선순위 제안
가장 먼저 잡을 3가지만 꼽자면 (1) 모바일 반응형 CSS 추가, (2) 입력값 검증 및 안내 메시지 개선, (3) 시나리오 Best/Worst 가정치에 대한 설명·편집 UI 제공입니다. 이 세 가지만 보완해도 "이사회/스테이크홀더에게 보여주는 도구"라는 포지셔닝이 훨씬 탄탄해질 거예요.
